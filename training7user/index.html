What Are AI Agents?

Concept AI Agents Defined ▼

An AI Agent is software that perceives its environment, reasons about a goal, and takes actions — autonomously or with human guidance — to achieve a defined outcome.

Key Idea: AI Agents are not just chat interfaces — they are goal-oriented systems that can perceive, reason, and act.

  1. P Perceive — Reads data from email, SharePoint, Teams, databases, or external systems
  2. R Reason — Uses an LLM to interpret instructions, context, and goals
  3. A Act — Sends messages, updates records, calls APIs, generates documents
  4. L Learn — Improves with feedback, memory, and grounding in your knowledge

Part 2 — Hands-On (~30 min)

Exercise 1 — Explore Pre-Built Agents

Exercise Discover Pre-Built Agents in Microsoft 365 ▼

Goal: Discover what pre-built agents exist in Microsoft 365, interact with at least two, and document your observations.

You will need: Microsoft 365 login with Copilot license • Edge or Chrome browser

  1. 1 Open Microsoft 365 at m365.cloud.microsoft/chat and sign in with your work account.
  2. 2 Open Copilot Chat. Ask it to summarize a document or draft an agenda for a team meeting.
  3. 3 Open the Agents/Agent Store experience and browse available agents. Click the agent gallery icon (top right of the chat bar).
  4. 4 Select one pre-built agent relevant to your work (e.g., Researcher, Analyst, or SharePoint Agent). Launch it.
  5. 5 Ask the agent 3 questions that relate to your day-to-day tasks. Note how it responds.
  6. 6 Try a second agent. Compare its behavior, tone, and capabilities to the first.

Exercise 2 — Build a Declarative Agent

Exercise Create Your First Agent ▼

Goal: Create your first declarative agent, ground it with a knowledge source, and test it with real questions.

You will need: Microsoft 365 login • Edge or Chrome • Access to a SharePoint site or sample document (provided) • Copilot Studio access (trial or licensed)

  1. 1 Open Agent Builder: Go to copilot.microsoft.com and click the sparkle/agent icon, then “Create an agent.”
  2. 2 Click “Create an agent” to open the configuration panel.
  3. 3 Name your agent — choose something relevant (e.g., “Franchise FAQ Agent,” “Guest Recovery Helper,” “Drive-Thru Coach”).
  4. 4 Write your system instructions. Tell the agent: who it is, what it helps with, its tone, and any constraints. Example: “You are a helpful franchise operations assistant. You answer questions from restaurant managers about policies, procedures, and brand standards. Always be concise and practical.”
  5. 5 Add a knowledge source using the options available in your tenant (SharePoint site or uploaded document) that supports your scenario.
  6. 6 Test your agent in the right-hand preview pane. Ask it 3 questions a real user might ask.
  7. 7 Refine the instructions based on the responses you see. Note what you changed and why.

Framework Ideation Worksheet ▼

Use this framework to identify the right agent scenario for your team:

  1. Pain Point: What process is painful, slow, error-prone, or repetitive?
  2. Who is Affected: Which role, team, or brand experiences this most?
  3. Current Solution: How is it handled today? (manual process, spreadsheet, email chain, phone calls, nothing…)
  4. What Would an Agent Do Differently: How would an agent perceive the situation, decide what to do, and act on it?
  5. Agent Type: Pre-built / Declarative / Custom engine
  6. Priority Score: Rate on three dimensions (1–5 each):
    • Business Impact (1 = Low, 5 = Critical)
    • Feasibility (1 = Hard, 5 = Easy)
    • Speed to Value (1 = Months, 5 = Days)
    Max: 15 points. Higher score = better fit for today’s build.

  • ☐ Knowledge source connected and indexed
  • ☐ Agent tested with at least 5 representative questions
  • ☐ Edge cases and off-topic questions tested

Scenario Starter Pack

Not sure where to start? Use these scenarios as inspiration or adapt them to your team.

1 Franchise Performance Agent ▼

Monitors KPI dashboards across brands. Alerts franchise managers when a location falls below thresholds for sales, speed of service, or guest satisfaction.

2 Guest Recovery Agent ▼

Handles inbound guest complaint messages from email or social channels. Triages severity, drafts a response, and escalates high-risk cases to the Regional Operations Manager.

3 New Hire Onboarding Agent ▼

Guides new restaurant team members through Day 1–30 onboarding. Answers policy questions, delivers training reminders, tracks completion, and escalates issues to the GM.

4 Supply Order Agent ▼

Monitors inventory levels against par standards. Detects low-stock situations, recommends reorder quantities based on projected sales volume, and initiates purchase orders with suppliers.

5 Marketing Campaign Brief Agent ▼

Generates brand-consistent campaign briefs for local store marketing. Incorporates brand guidelines based on the franchise region and brand identity.

6 Food Safety Audit Agent ▼

Guides shift managers through food safety checklists (temperature logs, holding times, cleaning schedules). Flags non-compliance, generates corrective action tickets, and logs audit results.

Build Tips & Best Practices

Tips 8 Tips for Great Agents ▼
  1. S Start Simple — A working MVP beats an unfinished complex agent every time.
  2. I Instructions Matter — Be specific about persona, tone, scope, and what to avoid.
  3. K Quality Knowledge — Use clean, well-structured docs as your knowledge source.
  4. T Test Adversarially — Ask off-topic questions and edge cases.
  5. B Business Value First — Tie your agent to a real outcome. Quantify if possible.
  6. D Document As You Go — Note design decisions for the demo.
  7. G Think About Guardrails — What should the agent NOT do?
  8. C Collaborate — Ops people know the pain points; tech people know the tools.

Glossary

Terms Key Vocabulary ▼
Term Definition
Agent An AI assistant that helps users achieve goals by answering questions and/or taking actions. Some agents can be configured to run proactively or automate actions depending on type and configuration.
Declarative Agent A Copilot customization defined by instructions, personality, and knowledge — no code required. Built with Agent Builder or Copilot Studio.
Autonomous Agent An agent configured to run proactively (e.g., on a schedule or event) and take actions based on triggers, where supported and permitted by policy.
Trigger The event or condition that initiates an agent’s workflow (e.g., form submission, schedule, Teams message, data threshold).
Action A discrete task performed by an agent — querying a database, generating a document, sending an email, or calling an API.
Connector A pre-built integration that allows agents to interact with external systems (e.g., SharePoint, Dataverse, Salesforce, ServiceNow).
Grounding The process of connecting an agent to specific, authoritative data so its responses are accurate and contextually relevant.
Knowledge Source A data repository connected to an agent — SharePoint sites, uploaded files, or websites.
System Prompt The foundational instructions that define an agent’s persona, purpose, constraints, and behavior. Called “system instructions” in Agent Builder.
Copilot Studio Microsoft’s low-code platform for building, customizing, and managing AI agents and copilot experiences.
Agent Builder An in-product experience in Microsoft 365 Copilot for creating declarative agents using instructions and selected knowledge sources.
