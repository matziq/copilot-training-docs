Section 1

The Art of Prompting

Key Concept What Is a Prompt? ▼

Prompts are how you ask Copilot to do something for you — like creating, summarizing, editing, or transforming content. Think about prompting like having a conversation. Use plain but clear language and provide context, just as you would with a human assistant.

Key Concept 1. Tell Copilot What You Need ▼

The first step is simply telling Copilot what you need. Here are six common use cases:

Use Case Example Prompt
Learn about projects & concepts “What are the main considerations for a new system launch?”
Edit text “Check this product launch rationale for inconsistencies.”
Transform documents “Transform this FAQ doc into a 10-slide onboarding guide.”
Summarize information “Write a session abstract of this [uploaded presentation].”
Create engaging content “Create a value proposition for a new product.”
Catch up on missed items “Provide a summary of the latest product announcements for Copilot.”

Section 2

Prompt Ingredients – GCSE

Framework The GCSE Framework ▼

To get the best response, focus on four key elements when phrasing your prompts:

Here is a mnemonic to help you remember the four elements.

Element What It Means Question to Ask
Goal What response do you want from Copilot? “What do I need?”
Context Why do you need it and who is involved? “What’s the background?”
Source Which information sources or samples should Copilot use? “What data should it reference?”
Expectations How should Copilot respond to best meet your expectations? “What format, length, or tone?”
Example GCSE Building Blocks – Progressive Example ▼

Watch how each GCSE element improves the prompt:

Elements Used Prompt
G only Generate 3–5 bullet points
G + C Generate 3–5 bullet points to prepare me for a meeting with Client X to discuss their Copilot launch.
G + C + S Generate 3–5 bullet points to prepare me for a meeting with Client X to discuss their Copilot launch. Use best practice guidance from the Microsoft Adoption site.
G + C + S + E Generate 3–5 bullet points to prepare me for a meeting with Client X to discuss their Copilot launch. Use best practice guidance from the Microsoft Adoption site. Please use simple language so I can get up to speed quickly.
Exercise GCSE in Practice – Job Description ▼

Try the following two prompts using Copilot Chat and compare the responses you get:

Prompt 1 – Goal Only
Write a job description for a senior project manager
Prompt 2 – Full GCSE
Generate a comprehensive job description for a senior project manager focused on technical project management for consumer electronic hardware. This role is urgent, and the candidate will join a dynamic team. Reference our company’s standard specs and industry norms. The description should be concise, max two pages, including responsibilities, qualifications.
GCSE Element Prompt 2 Mapping
Goal Generate a comprehensive job description for a senior project manager
Context Focused on technical project management for consumer electronic hardware; urgent role, dynamic team
Source Reference our company’s standard specs and industry norms
Expectations Concise, max two pages, including responsibilities and qualifications

What differences do you notice? How does specificity change the quality of the output?

Section 3

Iterating & Expanding Your Prompts

Example Good → Better → Best ▼

Scenario: You are an HR Manager who needs to prep for an upcoming candidate interview for a new AI Engineer position within the IT department.

Good
Help me prepare for an upcoming candidate interview for a new AI Engineer role in our IT department by creating a list of questions
Better
I’m an HR Manager who needs to prepare for a candidate interview for our new AI Engineer role in IT. Please prepare a list of appropriate questions for the candidate, so that I look prepared during the interview
Best
I’m an HR Manager who needs to prepare for a candidate interview for our new AI Engineer role in IT. Please review the candidate’s resume here: /[resume file] and the position detail here: /[position doc] and provide me a comprehensive list of questions that I can use to determine the candidate’s fit against the position
Level GCSE Elements Present
Good Goal only
Better Goal + Context + Expectations
Best Goal + Context + Source + Expectations
💡 Key Insight
Notice that “Best” adds Source — the game-changer that grounds Copilot in your real data rather than generic knowledge.
Key Concept Expand the Purpose of the Prompt ▼

Make your prompts more impactful by thinking about how Copilot fits into your entire workflow, not just a single task.

Question to Ask Yourself What It Means
Why am I using this prompt? Understand the context for this task
What will I do with the output? Will I send it to someone? Transform it? Feed it into another step?
Can I extend Copilot’s remit? Can I ask Copilot to do a bigger task to automate more work?
Exercise Extending the Prompt’s Purpose – Meeting Follow-Up ▼

We’re all familiar with creating a list of actions from Teams meetings — but why are you doing it? If it’s to send to someone else, why not get Copilot to write the email too?

Try This – Single Task
Recap the /[Meeting Name] meeting creating a table for action items, owners and due dates.
Try This – Extended Purpose
Write a follow up email to the attendees of the /[Meeting Name] meeting with a table showing decisions made, another showing actions and owners and lastly a list of considerations for the next meeting
💡 Key Insight
Don’t just ask Copilot for the intermediate step. Ask it for the end product you actually need.
Exercise Practice – Provide Specific Instructions ▼

Copilot is built to take action, so telling Copilot what “to do” or what “not to do” can be very effective.

Try This – Simple
Write an overview of how Microsoft 365 Copilot can help improve productivity in enterprises. Generate 3 bullet points.
Try This – Specific Instructions
Summarize the latest Microsoft AI announcements for a keynote speech on stage at Ignite, Microsoft’s annual conference. Do not include any announcements from before January 2025 and ensure to highlight the impact of these innovations.

Section 4

Conversational Prompting

Key Concept Keep the Conversation Going ▼

Following up on your prompts helps you collaborate with Copilot to gain more useful, tailored responses. You don’t need to get it perfect on the first try — iterate.

Use Case How to Keep the Conversation Going
Generating content ideas Lead with broader requests, then give specific details about the content
Enabling insightful meetings Request a meeting recap, then ask for more information about what you should know
Storytelling assistance Ask Copilot to write a story, then guide it with more specific, relevant details
Gaining insights Ask for a summary of a specific file, then ask relevant questions for deeper insights
Translating languages Ask Copilot to translate a sentence, then ask for more context or a regional dialect
Solving technical problems Present a technical problem, then narrow it down or ask for step-by-step guidance
Exercise Practice – Conversational Refinement ▼

Let’s refine the output with a conversational approach.

Starting Prompt
Write an overview of how Microsoft 365 Copilot can help improve productivity in enterprises. Generate 3 bullet points.

Now refine it! Try follow-up prompts like:

  • “Make these more specific to finance teams”
  • “Add a real-world example for each bullet”
  • “Rewrite in a more executive-friendly tone”

Section 5

Use Existing Materials as Templates

Key Concept Give Copilot Examples of Good ▼

Providing a sample sentence, paragraph, or existing document to Copilot offers a guideline and enhances the quality of the output. This ensures the result is formatted in a familiar and ideal way, minimising the time you need to spend reviewing and editing.

Exercise Practice – Using a Template Document ▼
Try This Prompt
Create me a proposal for my upcoming sales call with my customer, using /[Business Proposal Template.docx] as a template
💡 Key Insight
Referencing an existing document as a template is one of the most powerful “Source” techniques in the GCSE framework. It gives Copilot a concrete example of what “good” looks like.

What Are Agents?

Key Concept Copilot vs. Agents ▼

While every employee has a Copilot to transform individual productivity, every business process can be transformed by agents.

Copilot Agents
Personal AI assistant for every employee Purpose-built AI that automates business processes
Responds to your prompts in real time Follows defined instructions, knowledge, and actions
General-purpose across M365 apps Tailored to a specific role, task, or workflow
💡 Key Insight
Everything you’ve learned about writing great prompts applies directly to writing agent instructions. The GCSE framework is the foundation for configuring agents.
Key Concept Components of an Agent ▼

Instructions define the agent and tailor its experience to business requirements and expected outcomes.

Component What It Does
Purpose What this agent is and its goals
Guidelines General directions, tone, rules and restrictions
Skills Granular tasks with step-by-step workflow, handled by the LLM
Example Corporate Q&A Agent – Instruction Breakdown ▼

To get the best user experience, provide as many details as needed to minimise the user’s effort in prompting your agent:

Component Example Instruction
Purpose You are Corporate Q&A agent that helps to share corporate knowledge.
Guidelines You answer user’s questions using only the company knowledge base and public documentation. Say “I do not know” if there are no relevant results.
Be nice and polite.
Acronym CE means [Company Electronics].
Skills You should answer the questions in the following format:
Question: inferred user question in bold
Answer: results rephrased to be more relevant for user query
💡 Key Insight
This maps to GCSE: Purpose = Goal, Guidelines = Context + Expectations, Skills = Source + Expectations.

Section 8

Prompting Best Practices

Reference 7 Prompting Best Practices ▼
# Best Practice What It Means
1 Prompt length and clarity matters Be specific but concise. Aim for 25–30 words instead of the average 7–9.
2 Break down complex problems Don’t cram everything into one prompt. Use a step-by-step approach for clarity and depth.
3 Use persona-based prompting Ask Copilot to “act as” a specific role for tailored guidance.
4 Ask Copilot what it needs Instead of guessing, ask Copilot what information you should provide.
5 Take a holistic approach Think about entire processes, not just individual tasks.
6 Be curious Experiment with different prompt structures and approaches.
7 Test and refine Iterate based on results. Small tweaks can make a big difference.

Try This Prompt

Wrap-Up

Key Takeaways

Summary What to Remember ▼
  • Use GCSE — Goal, Context, Source, Expectations — every time you write a prompt
  • Iterate — your first prompt is a starting point, not the final answer
  • Keep it conversational — follow up, refine, and build on responses
  • Use templates — reference existing documents for consistent, high-quality output
  • Share what works — use the Prompt Gallery to save and share prompts with your team
  • Experiment — the best way to learn is to try different approaches
