Section 5

Build Phase Coaching Guide

Schedule Check-in Schedule ▼
Time Check-in Goal
1:10 PM Quick round — every team has a finalized canvas (not still planning)
1:30 PM Deeper check-in: “What is your agent doing right now?” (not “How are you doing?”)
2:00 PM “Are you on track for a demo at 3:15?” Nudge any team to narrow scope if needed
2:30 PM “Stop building. Start preparing your demo.” Enforce this firmly.
Coaching Coaching Prompts ▼

When a team is stuck

  • “If I were the user, what would I type to start a conversation with your agent?”
  • “What is ONE thing your agent absolutely must do? Just build that first.”
  • “What is the simplest version of this that still demonstrates value?”

When a team is overcomplicating

  • “What if your agent only did one specific action? Would that still be useful?”
  • “A narrow scenario done well is worth more than an ambitious one half-built.”

When a team finishes early

  • “Can you add a second knowledge source to improve accuracy?”
  • “What questions is your agent getting wrong? Fix that with better instructions.”
  • “Can you improve the demo flow so value is obvious in the first 30 seconds?”
Troubleshooting Common Technical Issues ▼
Issue Likely Cause Resolution
Agent Builder not loading License not provisioned or wrong tenant Check microsoft365.com/admin; try Edge browser
Knowledge source not returning results Still indexing (2–5 min) Wait and retry; have a fallback document ready
Copilot Studio loading slowly Browser cache or VPN Clear cache, disable VPN, try incognito mode
Agent giving wrong answers Instructions too vague Help refine system prompt to be more specific
Cannot log into Copilot Studio Wrong tenant or no license Confirm tenant URL; check license in M365 Admin
WiFi dropping Guest network blocking M365 Switch to personal hotspot immediately
Team completely stuck Scope too broad, no clear user Name ONE user with ONE problem. Build for that person only.
⚠️ Most Common Hackathon Failure Mode
The most common reason teams fail to demo is trying to build too much.
A working agent that does ONE thing well beats a broken agent that tries everything.
If a team has more than 3 topics in Copilot Studio at 2:00 PM, intervene.

Section 7

FAQ & Troubleshooting

FAQ What if an attendee does not have a Microsoft 365 Copilot license? ▼

They can participate in Copilot Studio exercises (which has a trial) and team up with a licensed colleague for Agent Builder. Pair them intentionally at team formation so they are never blocked.

FAQ What if Copilot Studio is slow or unresponsive? ▼

Usually a browser issue. Try: (1) clear browser cache, (2) use incognito/private mode, (3) switch from Chrome to Edge, (4) disable VPN. Edge is the most reliable browser for M365 tools.

FAQ What if a team finishes building before 3:00 PM? ▼

Use coaching prompts: add a second knowledge source, improve instruction quality, practice and tighten the demo flow, or begin drafting a production deployment plan.

FAQ What if a team is completely stuck? ▼

Scope is too broad. Walk them through this: “Name one person. Name one thing that person asks repeatedly. Build an agent that answers only that one thing.” This always unlocks them.

FAQ What if attendees are skeptical about AI? ▼

Do not argue. Say: “Your skepticism is healthy — we are going to let the demo speak for itself. Try building one thing, and then tell me what you think.” Skeptics who build something often become the loudest advocates.

FAQ What if the WiFi fails completely? ▼

Switch to personal hotspot immediately. If all connectivity fails: pause the build phase, run group paper-based scenario planning using the canvas, and reschedule the actual build as a virtual session within 2 weeks.

FAQ What if no one wants to go first in demos? ▼

Prep a volunteer in advance — briefly ask one confident attendee over lunch if they will present first. A strong opening demo sets the tone and encourages everyone else.

FAQ What if a team’s agent does not work during the demo? ▼

Normalize it: “This is exactly what happens in real pilots, and it is why we demo early.” Ask them to walk through what the agent is supposed to do and what they would fix. Intent and design are as valuable as a working demo in this context.

Ready-to-use agent configurations for Agent Builder (declarative agents). Each agent includes instructions, knowledge sources, and starter prompts you can copy directly.

M365 Copilot Premium — Agent Builder supports custom instructions, SharePoint/OneDrive knowledge sources, and starter prompts. It does not support external connectors, automated triggers, or proactive notifications — those require Copilot Studio.

When a user picks a writing task, deliver polished output that matches [Company Name]'s voice and style.
