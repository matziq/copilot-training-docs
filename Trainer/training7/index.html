Module 1 — 9:30–11:00 AM

AI Agents in Microsoft 365

🎯 Learning Outcomes
Understand agent fundamentals, the agent spectrum, pre-built vs. declarative vs. custom agents, agent architecture, and real-world business value.
Concept What Are AI Agents? ▼

An AI Agent is software that perceives its environment, reasons about a goal, and takes actions — autonomously or with human guidance — to achieve a defined outcome.

  1. P Perceive — Reads data from email, SharePoint, Teams, databases, or external systems
  2. R Reason — Uses an LLM to interpret instructions, context, and goals
  3. A Act — Sends messages, updates records, calls APIs, generates documents
  4. L Learn — Improves with feedback, memory, and grounding in your knowledge
📝 Trainer Note
Emphasize the difference between agents and standard Copilot Chat: agents are goal-oriented systems that can take actions, not just answer questions. Use the PRAL framework to make it memorable.
Concept The Agent Spectrum — Low to High Complexity ▼
Complexity Agent Type Platform Key Capabilities
Low Pre-Built Agents Microsoft 365 Copilot Chat, Researcher, Analyst, SharePoint Agents, Facilitator Agent — zero configuration
Medium Declarative Agents Copilot Studio Lite / SharePoint Custom persona & tone, scoped knowledge sources, no-code configuration, controlled topic scope
High Custom & Autonomous Agents Copilot Studio Autonomous workflows, multi-step reasoning, external connectors, API calls, approval & escalation

Talking Points Declarative Agents — Copilot Studio Lite & SharePoint ▼

What Are Declarative Agents?

  • Custom Copilot persona with a specific name, role, and tone
  • Scoped to specific knowledge sources (SharePoint, files, URLs)
  • Built in minutes using Copilot Studio Lite — no coding required
  • Deployed in Microsoft Teams, SharePoint, or Copilot Chat
  • Configured via natural language instructions

When to Use:

  • You have docs/SOPs that employees constantly ask about
  • You want a scoped Q&A bot without coding
  • HR policy, training materials, compliance guides
📝 Trainer Note
Use a concrete example: a “Franchise Ops Assistant” scoped to a SharePoint site that answers questions about brand standards, equipment specs, and operational checklists. Deployed in Teams for all field staff.

Module 2 — 11:15 AM–12:30 PM

Hands-On Exercises

🎯 Learning Outcomes
Explore pre-built agents in Microsoft 365 and build a declarative agent using Copilot Studio Lite. Two structured exercises to build confidence before the hackathon.
Exercise 1 Explore Pre-Built Agents (~30 min) ▼

Goal: Experience how pre-built agents handle real tasks — no setup required.

  1. 1 Sign In — Go to m365.cloud.microsoft/chat and sign in with your Microsoft 365 account
  2. 2 Try Copilot Chat — Ask it to summarize a document, then ask it to draft an agenda for a performance review meeting
  3. 3 Open Researcher Agent — In the left panel, click Agents → Researcher. Prompt: “Research trends in drive-thru technology for 2025–2026.”
  4. 4 Try the Analyst Agent — Click Agents → Analyst. Paste in the sample weekly sales table (provided). Ask for a daypart breakdown and chart.
  5. 5 Explore SharePoint Agent — Navigate to the sample SharePoint site, click the Copilot icon, and ask a question about the content.
  6. 6 Reflection — Note what impressed you and where you see gaps. Each participant shares 1 insight in the debrief.
📝 Trainer Note
Walk through Step 2 live on the projector, then let participants proceed independently through Steps 3–6. Circulate to help anyone who can’t find the Agents panel. Have the sample sales data ready to share via Teams chat.
Exercise 2 Build a Declarative Agent (~45 min) ▼

Goal: Build a scoped Copilot agent grounded in your organization’s knowledge — in under 45 minutes.

  1. 1 Open Copilot Studio — Go to copilotstudio.microsoft.com. Click Create → New Agent. Select “Copilot Studio Lite” (no-code).
  2. 2 Name & Describe — Name your agent (e.g., “Ops Assistant”) and write a short description of what it does.
  3. 3 Write Instructions — System prompt example: “You are a helpful assistant for operations. Answer questions about brand standards, policies, and operational procedures. Keep responses concise and professional.”
  4. 4 Add Knowledge Source — Click Add knowledge → SharePoint. Paste the sample SharePoint URL provided. Wait for indexing to complete (2–3 min).
  5. 5 Test Your Agent — Use the test panel on the right. Ask real questions a user would ask. Refine instructions based on the response.
  6. 6 Optional: Publish to Teams — Click Publish → Microsoft Teams. Share the link with a colleague to test cross-device.
📝 Trainer Note
Walk through Steps 1–3 live before letting participants proceed. Emphasize that the system prompt is the soul of the agent — spend time helping participants write clear, specific instructions. Share the sample SharePoint URL in Teams chat for easy copy-paste.
Debrief Exercise Debrief & Discussion ▼

Discussion Questions:

  1. What surprised you most about the pre-built agents?
  2. Where did you see the biggest potential for your organization?
  3. What limitations or gaps did you notice?
  4. Which agent felt most relevant to your day-to-day work?
  5. What questions do you have before the Hackathon?

Framework Ideation Framework — Find Your Best Agent Scenario ▼

Use this 3-step thinking process to identify a high-value agent scenario:

  1. 1 Pain Point — What frustrates your team daily? Same questions repeated, reports that take hours, new hires who can’t find policies, complaints not routed consistently.
  2. 2 Process — What repeatable process underlies it? Q&A lookup → knowledge base. Data aggregation → reporting. Document search → onboarding. Intake & routing → resolution.
  3. 3 Agent Type — Which agent fits? Q&A → Declarative Agent. Data analysis → Analyst Agent. Searching docs → SharePoint Agent. Multi-step workflows → Copilot Studio.

TIP: The best scenarios have a clear before/after — pick something where the agent saves real time or reduces errors.

📝 Trainer Note
Give teams 15 minutes for ideation. Circulate and help them narrow down to one scenario. Approve any custom scenarios before build starts.

Build Tips & Best Practices

Tips 8 Tips for Building Great Agents ▼
  1. S Start Simple — Build the smallest version first. A working MVP beats an unfinished complex agent every time.
  2. I Instructions Matter — Spend time on your system prompt. Be specific about persona, tone, scope, and what to avoid.
  3. K Quality Knowledge — Garbage in, garbage out. Use clean, well-structured SharePoint docs or files as your knowledge source.
  4. T Test Adversarially — Ask your agent off-topic questions and edge cases. Make sure it stays in scope and fails gracefully.
  5. B Business Value First — Always tie your agent to a real outcome: time saved, errors reduced, experience improved. Quantify if possible.
  6. D Document As You Go — Note your design decisions, what worked, and what you’d change. This helps during the demo and pilot phase.
  7. G Think About Guardrails — What should the agent NOT do? Define restrictions in your system prompt to prevent scope creep and misuse.
  8. C Collaborate — Mix roles in your team. Ops people know the pain points; tech people know the tools. Both perspectives matter.

Demo Day — 4:00 PM

Demos & Judging

Format 5-Minute Demo Script ▼
  1. 0:00 The Problem (30 sec) — State the pain point in one sentence. Who feels it? How often?
  2. 0:30 Meet Your Agent (1 min) — Show the agent name, type, and knowledge sources. Explain your system prompt in plain language.
  3. 1:30 Live Demo (2 min) — Show 2–3 real interactions. Narrate what the agent is doing and why.
  4. 3:30 Business Value (1 min) — Quantify the impact: time saved, errors reduced, or experience improved.
  5. 4:30 What’s Next (30 sec) — If you had more time or resources, what would you add or improve?
📝 Pro Tips
  • Have a backup screenshot in case of tech issues
  • Demo with real-sounding questions, not generic ones
  • Speak to the specific business use case
  • It’s OK if it’s not perfect — show the potential

Creativity & Innovation Unexpected, inspiring, opens new possibilities Solid idea, some creative elements Expected or derivative solution
