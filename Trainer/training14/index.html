Work IQ — What Copilot Knows About You

Exercise Microsoft 365 Work IQ ▼

Where to use: M365 Copilot Chat (m365copilot.com)

Prompt
What do you know about me? Tell me how I work, what I spend my time on, what topics, cases or projects I focus on, and who I collaborate with most frequently. Include any patterns you've noticed about my communication style and work habits.
📝 Trainer Tip

Section 3

Comparing Documents

Exercise Comparing Documents in M365 Copilot ▼

Where to use: M365 Copilot Chat (m365copilot.com)

This exercise compares two versions of a Master Supply Agreement to identify material changes. Upload both documents or use the / file picker to reference them from SharePoint/OneDrive.

Prompt
Compare /Master_Supply_Agreement_V1_Original.docx and /Master_Supply_Agreement_V2_Revised.docx. Identify every material change between the original and revised versions. For each change, note the section or clause number, describe what was modified, and assess the legal significance. Flag any changes that increase risk exposure, alter indemnification terms, modify liability caps, or change dispute resolution provisions.
📝 Trainer Tip
This is a signature moment. The comparison highlights material differences including: expanded ESG/sustainability requirements, new indemnification caps, enhanced data privacy provisions (CCPA), extended insurance requirements, and new termination triggers for ESG non-compliance. Let Copilot generate the analysis, then walk through 2–3 key findings to show how it identified substantive legal changes vs. cosmetic edits.

Tip: If using the / file picker, type the first few letters of the filename and select from the picker. Alternatively, upload both files directly.

Section 4

Copilot Researcher Agent — Deep Legal Analysis

Exercise Deep Legal Research with the Researcher Agent ▼

Where to use: M365 Copilot Chat (m365copilot.com) — Select Researcher

M365 Copilot Premium — Researcher agent requires a Copilot license

This exercise uses the Researcher agent to conduct a comprehensive legal analysis of the FDA’s Food Traceability Rule. The Researcher will search the web, compile findings, and produce a structured legal memorandum.

Prompt
Conduct a comprehensive legal analysis of the FDA's Food Traceability Rule (FSMA Section 204(d), 21 CFR Part 1, Subpart S), which established new traceability recordkeeping requirements for foods on the Food Traceability List. The federal compliance deadline is July 20, 2028. Analyze this regulation's specific impact on a major U.S. grocery retailer operating over 1,800 stores across 28 states under multiple regional banners. The company has set its own supplier traceability deadline of January 1, 2027 -- well ahead of the federal deadline.

Include in your analysis:
• Summary of the rule's key requirements for retail grocery operations, including the Food Traceability List, Critical Tracking Events, and Key Data Elements
• Specific compliance challenges for a multi-banner grocery retailer with complex, nationwide supply chains
• Relevant case law, legal challenges, and FDA enforcement actions related to FSMA traceability -- provide full legal citations
• Estimated compliance costs and penalties for non-compliance
• Recommended next steps for the company's legal and compliance teams

Format this as a professional legal memorandum suitable for review by in-house counsel.

After entering the prompt, select “Long” for the research depth. Say “Go ahead” or answer Researcher’s questions with your preferences.

📝 Trainer Tip
The Researcher agent typically takes 2–4 minutes to complete. Start it live and narrate what’s happening: “Researcher is searching for FSMA regulations, case law, FDA enforcement actions, and compliance guidance. It will compile everything into a structured legal memo.”

Have a pre-completed Researcher report open in another tab as a fallback. When the results arrive, highlight how it includes legal citations, compliance timelines, and actionable next steps — the kind of memo that would normally take a paralegal hours to compile.

Key talking point: Copilot can’t replace legal judgment, but it dramatically accelerates the research phase. The attorney still reviews, validates citations, and applies professional judgment.

Section 7

Building a Contract Review Agent

Exercise Build a Contract Review Agent in Agent Builder ▼

Where to use: Agent Builder at m365copilot.com

M365 Copilot Premium — Agent Builder requires a Copilot license

Follow these steps to create your own NDA Contract Review Agent:

Step Action Details
1 Create the Agent Open Agent Builder from m365copilot.com
2 Provide Basic Details Name: NDA Review Agent
Description: Reviews NDA contracts to ensure they follow Zava Contract Reference Standards.
3 Specify Agent Instructions Copy the instructions from the Agent Configuration document (see below)
4 Add Knowledge Source(s) Upload the Zava NDA Reference Standard v1.1 document from SharePoint or directly
5 Review and Test Test with the BluePeak NDA or HarborPoint Mutual NDA sample documents
📝 Trainer Tip
This is the centerpiece of the session for many audiences. Walk through the Agent Builder step-by-step. Build the agent live, paste the instructions, upload the knowledge document, then test with the BluePeak NDA. The agent should identify key deviations from Zava’s standard positions.

Key talking point: “This agent can now be shared with your entire legal team. Every NDA review starts with a consistent, standards-based first pass. The attorney still makes the final call, but the initial screening is done in seconds instead of hours.”
Reference Agent Configuration — Instructions to Copy ▼

Copy these instructions into the Agent Builder’s “Instructions” field:

Agent Instructions
## ROLE You are Zava Legal's NDA first-pass reviewer. Your job is to compare an uploaded vendor NDA to Zava's NDA Reference Standard (from Knowledge) and flag deviations. ## SCOPE - Do NOT approve/reject agreements. Provide an assistive review only. - Do NOT provide external legal advice or cite external law. - Use Zava NDA Reference Standard in Knowledge as the authority. If you can't find a standard, say so. - Do not invent missing clauses. If it's not in the NDA, mark it MISSING. - If key context is missing (mutual vs one-way, jurisdiction, purpose), ask up to 3 questions, then proceed with best-effort review. ## INPUTS YOU MAY RECEIVE - An uploaded NDA document (preferred) - Pasted NDA text (acceptable) - Minimal context from user (e.g., "vendor NDA, mutual") ## REVIEW STEPS 1) Identify basics: - Is it an NDA? Mutual or one-way? - Parties, purpose, term length, governing law/venue (if present) 2) Extract these NDA sections (use "Not found" if missing): - Confidential Information definition + exclusions - Use limitation / purpose restriction - Permitted disclosures (representatives / need-to-know) - Standard of care (how it must be protected) - Term (agreement term) - Survival (how long confidentiality lasts after termination/expiry) - Remedies (injunctive relief, penalties) - Return/Destruction requirements - Governing law / venue 3) Compare each section to Zava's NDA Reference Standard and tag: - OK = matches Zava standard - MISSING = required by Zava standard but absent - NEED ATTENTION = deviates from Zava standard or triggers a red flag 4) For every MISSING or NEED ATTENTION item: - Explain the issue in 1-2 sentences - State Zava's preferred position (from the standard) - If the standard lists a fallback, include it - If it's a red flag, label it "Escalate" ## OUTPUT FORMAT (ALWAYS) ### Executive Summary (max 4 bullets) - Overall posture (routine vs needs escalation) - Top 2-3 issues only ### Findings (by clause) For each clause: - Clause: - Tag: OK / MISSING / NEED ATTENTION - Issue: (1 sentence) - Zava Standard: (preferred + fallback if applicable) - Suggested fix: (short) ### Questions for the business (optional, max 3) - Only ask if needed to assess a deviation ## PRIORITY CHECKS (DON'T MISS THESE) - If survival language is absent → tag MISSING - If destruction requires deleting internal notes/analyses/derivatives → tag NEED ATTENTION + Escalate
Reference NDA Reference Standard — Knowledge Document Summary ▼

The Zava NDA Reference Standard v1.1 is uploaded as the agent’s knowledge source. Key sections include:

Section Key Points
Required Clauses Confidential Information definition, use limitation, standard exclusions, term & survival, governing law
Standard Positions Broad CI definition (includes derivatives), 3-year survival preferred, mutual NDAs when bilateral, injunctive relief for breach, need-to-know disclosure
Acceptable Fallbacks 2-year term (minimum), major U.S. commercial jurisdiction, written-only CI with clear marking, retention of compliance copies on return/destroy
Red Flags Unilateral NDA with Zava disclosing, sub-2-year terms, broad affiliate disclosure, mandatory destruction of internal notes, non-U.S. venue
Test Documents Sample NDAs for Agent Testing ▼

Use these sample NDAs to test the agent. Each has different characteristics that the agent should identify:

Document Type Key Features to Identify
BluePeak NDA One-way (Zava is Recipient) Vendor evaluation context, standard exclusions, injunctive relief clause, return/destroy provision, no explicit survival period
HarborPoint Mutual NDA Mutual Technology partnership context, 3-year term, 3-year survival, trade secret extension, Washington state governing law, compliance copy retention
📝 Trainer Tip
Start with the BluePeak NDA — it has a missing survival clause that the agent should flag as MISSING. Then test with HarborPoint, which is closer to Zava’s standard and should return mostly OK tags. The contrast between the two demonstrates the agent’s ability to differentiate quality.

The GCSE Framework

Framework Goal · Context · Source · Expectations ▼

The GCSE framework is the 4-part prompt structure that gets the best results:

Mnemonic: Giraffes Can See Everything

Element What It Means Legal Example
Goal What you want Copilot to do “Compare these two contract versions…”
Context Background: audience, purpose, constraints “…for a supply agreement between a retailer and a produce supplier…”
Source Where to pull data from: files, documents, or pasted text “…using /MSA_V1.docx and /MSA_V2.docx…”
Expectations Format, tone, length, output style “…flag changes that increase risk exposure or alter indemnification terms.”

Remember: There are no “wrong” prompts! Natural, conversational language works great. The more specific context you provide, the better Copilot’s response will be. Experiment and iterate — you can always refine your prompt based on the results.
